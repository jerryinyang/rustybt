    Story X2.3: P1 Code Quality Baseline¶

    Status¶

    Completed

    Story¶

    As a Software Engineer maintaining RustyBT's codebase quality, I want a clean code quality baseline with enforced linting, formatting, scoped type checking, and pre-commit hooks, so that the codebase maintains high standards, prevents quality drift, and provides a foundation for automated CI/CD quality gates.

    Acceptance Criteria¶

    Phase 1: Linting & Formatting¶

    1. Ruff Auto-Fixes Applied
    2. Run ruff check . --fix to auto-remediate fixable violations
    3. Repeat ruff check . --fix until no more auto-fixes available
    4. Manually fix remaining violations (invalid noqa directives, annotations, etc.)
    5. Configure McCabe complexity limit in pyproject.toml: max-complexity = 10
    6. Final ruff check . reports 0 errors (warnings acceptable if documented)

    7. Document ruff configuration changes in commit message

    8. Legacy Docstring Enforcement Strategy

    9. Identify legacy files with extensive docstring violations
    10. Add per-file ignores to [tool.ruff.lint.extend-per-file-ignores] for legacy files
    11. Ignore codes: D100 (module docstring), D101 (class docstring), D102 (method docstring) as needed
    12. New files (Epic 8+) maintain full docstring compliance

    13. Document strategy: "Legacy files exempted, new code strict" in CONTRIBUTING.md

    14. Black Formatting Applied

    15. Run black . to format all Python files
    16. Verify black --check . reports 0 files to reformat
    17. Line length configuration matches existing: 100 (check pyproject.toml)
    18. No functionality changes from formatting (verified via tests)
    19. Commit formatting changes separately from functional changes

    Phase 2: Type Checking¶

    1. Mypy Strict Enforcement on Scoped Modules
    2. Verify scoped strict modules in pyproject.toml: rustybt/exceptions.py, rustybt/utils/logging.py, rustybt/utils/error_handling.py
    3. Add Epic 8 modules to strict list: rustybt/analytics/*, rustybt/utils/secure_pickle.py
    4. Run python3 -m mypy (project config) → 0 errors for strict modules
    5. Legacy modules use relaxed settings via [[tool.mypy.overrides]]

    6. Document gradual typing migration strategy in docs/guides/type-hinting.md

    7. Type Stubs

    8. Verify dev extras include: pandas-stubs, types-networkx, types-requests, types-PyYAML
    9. Add missing stubs if identified during mypy execution
    10. Configure ignore_missing_imports = true for libraries without stubs: exchange_calendars, etc.

    11. Document stub availability in type-hinting guide

    12. Low-Hanging Type Annotations

    13. Fix rustybt/extensions.py annotations per gap analysis:
      • extension_args: dict[str, str] (or appropriate type)
      • custom_types: dict[str, type] (or appropriate type)
    14. Fix any top-level annotation errors that block mypy execution
    15. Run mypy to verify fixes resolve errors

    Phase 3: Pre-Commit Hooks¶

    1. Basic Quality Hooks Installed
    2. .pre-commit-config.yaml includes ruff hook with --fix arg
    3. .pre-commit-config.yaml includes black hook
    4. .pre-commit-config.yaml includes mypy hook (scoped to strict modules)
    5. Run pre-commit install and document in CONTRIBUTING.md
    6. Run pre-commit run --all-files passes successfully
    7. Hooks tested with sample commit (verify they run and block bad commits)

    Integration Requirements¶

    1. Existing Functionality Unchanged
    2. Full test suite passes after formatting/linting: pytest -m "not memory and not api_integration and not live and not ib_integration"
    3. Smoke test: bundle ingestion, backtest execution, report generation
    4. Manual verification: paper trading starts successfully

    5. No functionality changes from code quality fixes (pure style/type changes)

    6. Code Quality Enforcement Follows Existing Patterns

    7. Ruff configuration extends existing [tool.ruff] settings
    8. Black configuration matches existing line length and style
    9. Mypy overrides extend existing [[tool.mypy.overrides]] pattern

    10. Pre-commit hooks follow existing .pre-commit-config.yaml structure

    11. Development Workflow Integration

      • Pre-commit hooks run fast (< 5 seconds for typical commit)
      • Hooks provide clear error messages when blocking commits
      • IDE integrations work (VS Code, PyCharm support ruff/black/mypy)
      • Documentation guides new contributors through setup: pre-commit install

    Quality Requirements¶

    1. Changes Covered by Tests

      • Test suite passes with new code quality tools
      • Mypy strict modules have unit tests with good coverage
      • No new untested code introduced by fixes

    2. Documentation Updated

      • CONTRIBUTING.md documents pre-commit setup: pre-commit install
      • docs/guides/type-hinting.md updated with gradual typing strategy
      • docs/guides/type-hinting.md lists strict modules and migration plan
      • README.md updated with dev extras installation: uv sync -E dev -E test

    3. No Regression Verified

      • Full test suite passes: pytest -m "not memory and not api_integration and not live and not ib_integration"
      • Coverage remains ≥90% for core modules
      • Pre-commit hooks do not block legitimate commits

    Technical Implementation Guidance¶

    Ruff Configuration: 

    # pyproject.toml
[tool.ruff]
line-length = 100
target-version = "py312"

[tool.ruff.lint.mccabe]
max-complexity = 10

[tool.ruff.lint.extend-per-file-ignores]
"rustybt/data/bundles/*.py" = ["D100", "D101", "D102"]  # Legacy docstrings
"rustybt/gens/*.py" = ["D100", "D101", "D102"]

    Black Configuration: 

    # pyproject.toml
[tool.black]
line-length = 100
target-version = ['py312']

    Mypy Configuration: 

    # pyproject.toml
[tool.mypy]
python_version = "3.12"
warn_return_any = true
warn_unused_configs = true
disallow_untyped_defs = true

[[tool.mypy.overrides]]
module = [
    "rustybt.exceptions",
    "rustybt.utils.logging",
    "rustybt.utils.error_handling",
    "rustybt.analytics.*",
    "rustybt.utils.secure_pickle",
]
disallow_untyped_defs = true

[[tool.mypy.overrides]]
module = [
    "rustybt.data.*",
    "rustybt.gens.*",
    "rustybt.pipeline.*",
]
disallow_untyped_defs = false
ignore_errors = false

[[tool.mypy.overrides]]
module = [
    "exchange_calendars.*",
    "ccxt.*",
]
ignore_missing_imports = true
