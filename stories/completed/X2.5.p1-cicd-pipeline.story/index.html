    Story X2.5: P1 CI/CD Pipeline¶

    Status¶

    ✅ COMPLETED - All CI/CD pipelines functional and all issues resolved

    Story¶

    As a DevOps Engineer establishing automated quality gates, I want a comprehensive CI/CD pipeline with 6 workflow files covering code quality, zero-mock enforcement, security, testing, dependency security, and performance, so that all quality checks are automated, branch protection is enforced, and production deployments meet all standards.

    Acceptance Criteria¶

    CI/CD Workflow Files¶

    1. Code Quality CI Job
    2. Create .github/workflows/code-quality.yml
    3. BLOCKING: ruff check . (must pass with 0 errors)
    4. BLOCKING: black --check . (must pass with 0 files to reformat)
    5. BLOCKING: python3 -m mypy (must pass for scoped modules)
    6. BLOCKING: python scripts/check_complexity.py (McCabe ≤10 for all functions)

    7. Branch protection rules require this job to pass before merge

    8. Zero-Mock Enforcement CI Job

    9. Activate .github/workflows/zero-mock-enforcement.yml (prepared in Story X2.4)
    10. BLOCKING: python scripts/detect_mocks.py --strict (0 mock patterns)
    11. BLOCKING: python scripts/detect_hardcoded_values.py --fail-on-found (0 hardcoded values)
    12. BLOCKING: python scripts/verify_validations.py --ensure-real-checks (all validators work)
    13. BLOCKING: python scripts/test_unique_results.py (unique outputs)

    14. Branch protection rules require this job to pass before merge

    15. Security CI Job

    16. Create .github/workflows/security.yml
    17. BLOCKING: bandit -r rustybt -ll -i (0 High severity issues)
    18. BLOCKING: truffleHog --regex --entropy=False . (0 secrets detected)
    19. BLOCKING: detect-secrets scan (0 secrets detected)

    20. Branch protection rules require this job to pass before merge

    21. Testing CI Job

    22. Create .github/workflows/testing.yml
    23. Run unit tests: pytest -m "not memory and not api_integration and not live and not ib_integration" --cov=rustybt --cov-report=term --cov-report=html
    24. Verify coverage: Core modules ≥90%, financial modules ≥95%
    25. Run property-based tests: pytest -m "property" --hypothesis-profile=ci
    26. Upload coverage reports as artifacts

    27. BLOCKING: Coverage thresholds must be met

    28. Dependency Security CI Job (Weekly)

    29. Create .github/workflows/dependency-security.yml (scheduled: weekly)
    30. safety scan --json > safety-report.json
    31. pip-audit --format json > pip-audit-report.json
    32. python scripts/check_licenses.py (verify no GPL-licensed dependencies)
    33. Upload security reports as artifacts

    34. NON-BLOCKING: Creates issues for vulnerabilities but doesn't block merges

    35. Performance Regression CI Job (Main Branch Only)

    36. Create .github/workflows/performance.yml (on main branch after merge)
    37. python -m rustybt benchmark --suite backtest --output benchmark-results.json
    38. python scripts/check_performance_regression.py --threshold=0.20 --baseline=benchmark-baseline.json
    39. Fail if performance degrades >20%
    40. NON-BLOCKING: Creates issues for regressions but doesn't block merges

    1. CI Performance Optimized

      • All workflows run in parallel where possible
      • Total CI execution time < 12 minutes for typical PR

    File: .github/workflows/code-quality.yml

    name: Code Quality

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  code-quality:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'

      - name: Install uv
        run: |
          curl -LsSf https://astral.sh/uv/install.sh | sh
          echo "$HOME/.cargo/bin" >> $GITHUB_PATH

      - name: Install dependencies
        run: |
          uv sync -E dev -E test

      - name: Run ruff linting (BLOCKING)
        run: |
          uv run ruff check .

      - name: Run black formatting check (BLOCKING)
        run: |
          uv run black --check .

      - name: Run mypy type checking (BLOCKING)
        run: |
          uv run python3 -m mypy

      - name: Check code complexity (BLOCKING)
        run: |
          uv run python scripts/check_complexity.py --max-complexity 10

    Workflow 5: Dependency Security CI Job (Weekly)

    File: .github/workflows/dependency-security.yml

    name: Dependency Security

on:
  schedule:
    - cron: '0 2 * * 1'  # Monday 2 AM UTC
  workflow_dispatch:  # Allow manual trigger

jobs:
  dependency-security:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'

      - name: Install uv
        run: |
          curl -LsSf https://astral.sh/uv/install.sh | sh
          echo "$HOME/.cargo/bin" >> $GITHUB_PATH

      - name: Install dependencies
        run: |
          uv sync -E dev -E test

      - name: Run safety scan
        run: |
          uv run safety scan --json > safety-report.json || true

      - name: Run pip-audit
        run: |
          uv run pip-audit --format json > pip-audit-report.json || true

      - name: Check licenses (BLOCKING)
        run: |
          uv run python scripts/check_licenses.py

      - name: Upload security reports
        uses: actions/upload-artifact@v3
        with:
          name: security-reports
          path: |
            safety-report.json
            pip-audit-report.json

      - name: Create issue if vulnerabilities found
        if: failure()
        uses: actions/github-script@v6
        with:
          script: |
            github.rest.issues.create({
              owner: context.repo.owner,
              repo: context.repo.repo,
              title: 'Weekly Dependency Security Scan: Vulnerabilities Found',
              body: 'Automated dependency security scan found vulnerabilities. Check workflow artifacts for details.',
              labels: ['security', 'dependencies']
            })
